# Database ERD – CAD Gurukul

## Entity Relationship Overview

```
users (1) ──< (1) student_profiles
users (1) ──< (*) parent_details
users (1) ──< (*) assessments
assessments (1) ──< (*) assessment_questions
assessment_questions (1) ──< (*) assessment_answers
assessments (1) ──< (1) career_reports
career_reports (1) ──< (*) report_downloads
users (1) ──< (*) payments
payments (1) ──< (1) career_reports [unlocks]
admin_users (1) ──< (*) activity_logs
users (1) ──< (*) activity_logs
users (1) ──< (*) ai_sessions
```

## Table Descriptions

### users
Primary identity table for all student/parent users.
- Links to student_profiles, payments, assessments, activity_logs

### student_profiles
Extended intake form data collected during onboarding.
- One-to-one with users
- Stores academic, personal, and preference data

### parent_details
Optional parent information linked to a student user.

### assessments
An assessment session that a student starts.
- Has multiple questions and answers
- Results in a career_report

### assessment_questions
Individual questions generated by AI for a session.
- Ordered by sequence
- Has category/type tags

### assessment_answers
Student responses to assessment questions.
- Linked to question and assessment

### ai_sessions
Logs every call made to an AI provider.
- Tracks model used, tokens, cost, latency

### career_reports
Generated career guidance report (free or paid).
- Stores JSON report data
- Has access_level flag (free/paid)

### payments
Razorpay payment records.
- Links to user and report
- Tracks order, payment, and status

### report_downloads
Audit log of PDF downloads.

### admin_users
Separate admin authentication table.

### activity_logs
General audit trail for user actions.

### pricing_plans
Configurable pricing (editable by admin).

### contact_queries
Contact form submissions.
